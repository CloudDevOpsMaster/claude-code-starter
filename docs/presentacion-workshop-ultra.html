⚡ Comparación · Decisión

Vibe Coding vs SPEC DRIVEN

🌀 Vibe Coding

  • 💬 "Le pido y veo qué sale"
  • ⚡ Rápido para explorar
  • 🎲 Resultado impredecible
  • 🧪 Ideal para explorar ideas

✅ SPEC DRIVEN

  • 📐 "Diseño primero, código segundo"
  • ⏱️ Más lento (pero vale la pena)
  • 🎯 Resultado predecible
  • 📈 Alta escalabilidad

🎯 La pregunta: ¿Podría otro dev entender y extender lo que el agente generó en 3 meses?

➡️ Proceso · Evolución

La Transición

Vibe Coding → Aprender → Producir → SPEC DRIVEN
[explorar] · [aprender] · [producir] · [escalar]

No es uno u otro. Hemos visto que funciona así: exploras, aprendes, formalizas cuando va a producción.

🔧 Herramienta 2 · Ejercicio en chat

Skills — Patrones Reutilizables

¿Qué es un skill?

La forma estándar que ya usan para hacer algo. Lo documentan una vez, el agente la sigue siempre.

Ejemplos que ya existen:

  • Cómo agregan endpoints
  • Cómo escriben tests
  • Cómo documentan APIs
  • Cómo hacen code review
# Skill: agregar-endpoint ## Cuándo usar Al crear endpoint REST ## Pasos 1. Define método + ruta 2. Valida el input 3. Implementa lógica 4. Responde con formato 5. Escribe 3 tests

✅ Herramienta 5 · Estándar de Calidad

Definition of Done

Antes de mergear un PR generado por IA:

  • ✓ Tests pasan
  • ✓ Cobertura ≥ 70%
  • ✓ Linting OK
  • ✓ Review humano
  • ✓ Sin secrets
  • ✓ Docs actualizados
  • ✓ Autor entiende el código

🔑 Quien mergea es responsable. Siempre.
